Redesigning WIZ app to increase smart light users.

Smart lights for your daily living

Overview

Wiz connect is a mobile application that allows users to connect, control, and automate their smart Wiz light bulbs.

My Role

I conducted user research interviews to understand user needs, identify main pain points, and find ways to improve the app. I Iterated the interface design through 2 rounds of user testing, one A/B testing, and surveys.

The Problem

  • Users don’t use secondary functions because they have to dig deep in the app to be able to do tasks like change the color of the lights and set a timer.

  • Turn on and off the lights manually is an easy task to do with the current app but users are willing to do more than that.

  • Smart lights are difficult to get used to specially because the current features restrict the user to follow a tight daily life schedule.

Design Goals

Current app

Improve the color pallet

Employ brighter colors and modern UI elements in the interface visuals to make the app feel more exciting.

Improve distribution of the information

Organize the items in a new system to make simple functions like color picker faster to find.

Make the app adapt to the user

Make the current automation features more flexible by introducing functions like timer that fit better with users’ daily life.

User Test 1

Prototyping & testing the essentials

The first round of user testing consisted in asking eight potential users to do the following three tasks: first to turn on a light, next to change the color of the light and finally to set a 15 min timer on the light. All sessions were handed over zoom for 30min average. Here are the top 2 highlights from the user testing results:

  • 75% of the testers couldn’t find the on/off button in the room screen.

  • Light controls like color slider, brightness slider, and on/off button should go inside the thumb reachable area.

UI/UX Iteration 1

A/B Testing

Gathering feedback from users

Next steps for this project was to get feedback about how well the app communicates with users and how could this niew interface improve. For this assessment a 5-10 min A/B testing was prepared using google forms. I got around 15 responses full of notes and comments that I used to determine which design direction I should take. The following images represent a small section of the A/B testing results.

Q1. Which one do you think communicates the color of the light better?



AB testing details→

Key takeaways from the A/B testing:

  • 60% of the respondents think that version B communicates better the color of light.

  • Most people prefer big and evident light color indicators than a small and subtle detail, someone suggested making the light spectrum even bigger.

  • B design was positively taken as an ambient light.

UI/UX Iteration 2

Preference Test: Adding Color to the UI

Once most of the app had a great layout and structure, it was time to select a color pallet and bring excitement to the interface. I decided to do a quick preference test on instagram showing two versions of the UI; one with light theme and the other one with dark theme.
